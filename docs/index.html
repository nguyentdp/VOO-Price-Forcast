VOO 5-Day Forecasting with Linear Regression

This project predicts the VOO ETF price of 5-trading-day ahead. I use past daily trading data and simple patterns (trend, momentum, volatility) to train a Linear Regression model. The goal is to demonstrate a clean ML workflow, not to provide investing advice.

Educational project only. Not financial advice.
Topic: Linear Regression
Horizon: 5 trading days
Time-based train/test split
Outliers handled (winsorization + clipping)

Data (beginner-friendly)

The dataset is historical daily trading data. Each row is one trading day.

Instrument: VOO (S&P 500 index fund)
Fields: Open, High, Low, Close, Volume
Daily data from Kaggle

What “OHLCV” means

  • Open: price at the start of the day
  • High: highest price that day
  • Low: lowest price that day
  • Close: price at the end of the day
  • Volume: how much was traded that day
Why a 5-day horizon? Next-day movement is extremely noisy. Predicting a short multi-day window can be slightly more stable and easier to interpret in a portfolio setting.

Model & Results

Model: Linear Regression (a simple model that learns a weighted combination of features).

Baseline (why it matters)

To tell whether the model is actually useful, I compare it to a simple reference point called the no-change baseline. This baseline assumes the future price will be exactly the same as today’s price (i.e., 0% return). If the model can’t beat this, it isn’t adding meaningful predictive value.

Financial data is noisy, so improvements are usually modest. The main value of this project is a clear, reproducible ML process: feature engineering → fair split → baseline comparison → evaluation + plots.

Feature Engineering

Instead of feeding raw prices directly into the model, I convert the data into signals that represent trend, momentum, volatility, and trading activity.

  • Recent movement (momentum): price changes over the last 1, 2, 5, 10 days
  • Trend: whether today’s price is above/below a short moving average (MA ratio)
  • Volatility: how “bumpy” recent movement has been (rolling standard deviation)
  • Daily behavior: the day’s range (high−low) and the overnight gap (open vs prior close)
  • Trading activity: log(volume), volume changes, and a 20-day volume average
  • Outliers: winsorization + clipping so extreme days don’t dominate the model
  • Fair evaluation: time-based split to avoid look-ahead bias
Why Linear Regression? It’s a strong baseline model for supervised learning and is easy to explain: the forecast is a weighted combination of signals.

How to Reproduce

These commands run the model and regenerate the plots (Matplotlib).

1) Put the CSV here

data/VOO Stock Data.csv

2) Train + evaluate

python src/train.py

3) Make plots (Matplotlib)

python src/make_plots.py

4) Copy plots into the website folder

mkdir -p docs/assets
cp reports/figures/*.png docs/assets/
Note on datasets: Many Kaggle datasets should not be redistributed publicly. A clean approach is to keep the CSV out of GitHub and provide download instructions.

Local preview

python -m http.server 8000 --directory docs

Then open: http://localhost:8000/
