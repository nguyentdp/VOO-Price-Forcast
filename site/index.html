VOO 5-Day Forecasting with Ridge Regression

This project predicts the 5-trading-day ahead price movement of Vanguard S&P 500 ETF (VOO) using engineered technical features from historical OHLCV data. Performance is compared against a simple no-change baseline.

Topic: Linear Regression (Ridge)
Horizon: 5 trading days
Time-series split (no shuffle)
Outlier handling: winsorization + clipping
View Results Plots How to Reproduce

Dataset

Instrument: VOO
Fields: Open, High, Low, Close, Volume
Range: 2010-09-09 → 2023-10-31
  • Target: 5-day future return (converted into a future price prediction).
  • Train/Test: last 252 trading days used as test (≈ 1 year).
  • Baseline: predict future price equals today’s price (“no change”).
Why 5-day horizon? Next-day price is extremely noisy; a short multi-day horizon is often more stable and easier to interpret in a portfolio setting.

Feature Engineering

Instead of using raw price “as-is”, the dataset is transformed into signals that represent trend, momentum, volatility, and volume behavior.

  • Momentum / returns: ret_1, ret_2, ret_5, ret_10
  • Trend: ma_ratio_5 (price relative to 5-day moving average)
  • Volatility: vol_10 and vol_20 (rolling std of daily returns)
  • Market behavior: daily range (High-Low scaled by price) and gap (Open vs prior close)
  • Volume signal: log_vol, log_vol_diff, log_vol_ma_20
  • Outliers: winsorization on spiky features; clipping after scaling
  • Validation: time-based split to avoid look-ahead bias

How to Reproduce

  • Place your CSV at: data/VOO Stock Data.csv
  • Run training: python src/train.py
  • Generate plots: python src/make_plots.py (saves PNGs to reports/figures/)
  • Copy plots to site assets: mkdir -p docs/assets && cp reports/figures/*.png docs/assets/
Note: Many Kaggle datasets shouldn’t be redistributed in public repos. A clean approach is to keep the CSV out of GitHub and provide download instructions in your README.

Limitations & Next Steps

  • Financial time series are noisy; improvements over baseline are typically modest.
  • This model uses only technical OHLCV features (no macro/news signals).
  • Next steps: walk-forward validation, “up/down” classification, compare with tree models.
